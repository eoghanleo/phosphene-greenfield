## Extended terminology notes
The PRD references a vocabulary that spans simulation science, game economy design, and chat-native interface conventions. Terms like myrmecology, stigmergy, trophallaxis, and pheromone gradient are used to describe ant behaviors that can inspire discovery moments. This vocabulary supports content variety without requiring heavy narrative arcs.

Economic terms such as sink, faucet, scarcity loop, and pacing curve are used to describe how resources flow through the colony economy. These terms should be interpreted in the context of fairness: a sink should never be a paywall, and a faucet should never be tuned to coerce spending. When these terms appear in feature descriptions, they should be tied back to the economy ledger and trust metrics.

Experimentation terms such as baseline, delta, control, and variance refer to the strategist toolset. The benchmark arena uses these terms to describe comparisons between colony builds, while the experiment notebook logs parameter changes and outcomes. Consistent use of these terms ensures that strategists feel the tools are serious rather than gimmicky.

Observability terms such as telemetry, sampling, anomaly detection, and trace ID describe the measurement system that underpins validation. These terms exist so that requirements can be verified without ambiguity. When a metric is defined in the success section, it should map to a specific telemetry event and a clear query logic.

Chat interface terms such as conversational trail, in-thread pulse, and continuity indicator describe the mechanisms that keep the in-chat experience coherent. These terms should not be conflated with notifications or push messaging; they are designed to be subtle and context-preserving. The glossary ensures that new contributors use the same phrasing when describing these patterns.

Performance terms such as frame budget, render tick, and simulation step describe the runtime constraints of WebGPU and WebAssembly. These terms are used to keep the strategist experience credible; if performance terminology is ignored, the simulation may appear shallow or unstable. The glossary reinforces that performance is part of the user promise, not an implementation detail.

Trust terms such as receipt archive, rollback policy, and persistence checksum describe the explicit evidence shown to players. These terms should appear in UX copy and support playbooks so that the player sees consistent language. The goal is to reduce uncertainty and to make continuity feel tangible rather than abstract.

Accessibility terms such as comfort layer, calm mode, and readability band describe the UX features that lower cognitive load. These terms should be used consistently in the UI so that players can discover them without hunting in settings. The glossary maintains a consistent tone that aligns with the relaxed persona.

Governance terms such as data minimization, retention window, and privacy summary relate to how the product handles player data. These terms support transparency and compliance without adding heavy legal language. They should be used in the privacy summary so that players can understand data handling in plain terms.

Roadmap terms such as phase gate, milestone threshold, and validation sprint indicate that the product is evidence-driven. These terms describe how the team decides whether to proceed or pivot. The glossary makes it explicit that feature delivery is not a substitute for validated outcomes.
